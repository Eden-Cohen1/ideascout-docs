# ideascout-docs

Canonical documentation for [ideascout](https://github.com/Eden-Cohen1/ideascout), built with
[Docusaurus](https://docusaurus.io/) and deployed on [Vercel](https://vercel.com/). Content is
maintained by the Hermes docs agent; humans review and approve every change.

## Layout

- `docs/` — the canonical Markdown pages (this is the whole site; docs-only mode at `/`).
- `registry.json` — maps ideascout source areas → doc pages (was the Notion "Doc Registry").
- `proposals/ledger.jsonl` — append-only record of every proposal and its outcome.
- `docusaurus.config.js`, `sidebars.js`, `src/css/custom.css` — site config. The sidebar is
  autogenerated, so the agent never edits `sidebars.js`.

## How changes land

1. A PR merges in `ideascout` → GitHub webhook → Hermes `propose-docs`.
2. The agent opens a `proposal/pr-<n>` branch here with the drafted change and a **draft PR**.
3. Vercel builds a **preview** of that branch. The agent posts the preview + PR link to Slack.
4. After Slack approval, `publish-docs` marks the PR ready and merges it → Vercel production.

## Local development

```bash
npm ci
npm start          # dev server with hot reload
npm run build      # production build (what Vercel runs); fails on broken links
```

## Deploy (Vercel, one-time)

Import this repo in Vercel. It auto-detects Docusaurus (build `npm run build`, output `build`).
Set project env var `DOCS_SITE_URL` to the production domain, and (optionally) `DOCS_REPO`.
Preview deployments for pull requests are on by default — that is what powers doc previews.
